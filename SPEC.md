# 차량 제조 프로세스 기준 문서

## 1. 차량 제조 순서

### 1) 자동차 타입 선택
- 선택 가능 타입: **세단(Sedan)**, **SUV**, **트럭(Truck)**
- 현재 3가지 타입을 제작 가능하며, 향후 타입이 추가될 수 있다.
  - → 타입 목록은 고정된 if/elif가 아닌 확장 가능한 구조(Enum + 매핑)로 설계 필요.

### 2) 부품 선택
차량에 들어갈 부품을 각각 선택한다.
- **엔진 (Engine)**
- **제동장치 (Brake System)**
- **조향장치 (Steering System)**

### 3) 완성된 차량 테스트
선택한 부품이 해당 차량 타입에 사용 가능한지 검사한다.

## 2. 제한 조건 (호환성 규칙)

### 제한조건 1 — 부품 간 공통 호환성 (모든 브랜드/차량 타입에 공통 적용)
- 제동장치에 Bosch 제품을 사용했다면, 조향장치도 Bosch 제품을 사용해야 한다.
  (타사 제품과 호환되지 않는다.)
  - 이는 특정 브랜드에 국한된 규칙이 아니라, **모든 브랜드에 공통으로 적용되는 예시**이다.
    (즉 "Bosch 제동장치 ↔ Bosch 조향장치" 조합 요구는 브랜드 호환성 규칙의 한 예시이며,
    향후 다른 브랜드 쌍에 대해서도 동일한 형태의 공통 호환 규칙이 추가될 수 있다.)

### 제한조건 2 — 차량 타입별 부품 제한
- Continental은 Sedan용 제동장치를 만들지 않는다. (→ Sedan에 Continental 제품 사용 불가)
- 도요타는 SUV용 엔진을 만들지 않는다. (→ SUV에 TOYOTA 엔진 사용 불가)
- WIA는 Truck용 엔진을 만들지 않는다. (→ Truck에 WIA 엔진 사용 불가)
- Mando는 Truck용 제동장치(Brake System)를 만들지 않는다. (→ Truck에 Mando 제동장치 사용 불가)

### 정리
- **제한조건 1**: 차량 타입과 무관하게 부품(제동장치-조향장치) 간 브랜드 호환성을 검사하는 규칙.
- **제한조건 2**: 특정 차량 타입에 특정 제조사 부품(엔진/제동장치) 사용을 금지하는 규칙.
- 위 규칙 중 하나라도 위반하면 해당 조합은 **사용 불가(FAIL)** 처리한다.
- 모든 규칙을 통과하면 **사용 가능(PASS)** 처리한다.
